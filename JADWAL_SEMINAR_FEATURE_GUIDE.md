# 📅 Jadwal Seminar Feature - User Guide

## ✅ Implementation Complete

The jadwal seminar feature has been fully implemented with the following components:

### Backend (Python)
- **File**: `agent_ai/tools/jadwal_seminar.py`
- **Class**: `JadwalSeminarTools`
- **Features**:
  - Form generation with tanggal input, ruangan dropdown, durasi input

  - Value extraction and validation
  - Message formatting for backend

- **File**: `agent_ai/nodes/executor_node.py`
- **Action**: `generate_jadwal_seminar`
- **Features**:
  - Two-stage flow: form display → form submission
  - Automatic dosen context detection
  - Error handling and logging

## 🚀 How to Use

### Step 1: Start the System
Make sure PHP and Python servers are running:

```bash
# Terminal 1: PHP Server
cd "d:\semester 6\PROYEK AKHIR 3\Proyek-Kelompok-07-PA-3\ui laravel"
php artisan serve

# Terminal 2: Python Agent API
cd "d:\semester 6\PROYEK AKHIR 3\Proyek-Kelompok-07-PA-3\agent_ai"
uvicorn api:app --host 127.0.0.1 --port 8002 --reload
```

### Step 2: Open Application
Navigate to: **http://localhost:8000/ai-agent/kelompok**

### Step 3: Request Jadwal Seminar
Type in the chat:
```
"buatkan jadwal seminar"
```
or
```
"buat jadwal presentasi untuk kelompok"
```

### Step 4: Fill in the Form
A form will appear with three inputs:

**1. Tanggal Mulai Seminar**
- Input format: "15 mei 2026" or "10 juni"
- Supports Indonesian month names
- Auto-completes year if not provided

**2. Ruangan**
- Dropdown with all available rooms
- Select ONE room (all kelompok will use same room)
- Rooms: Auditorium, Ruang 1, Ruang 2, etc.

**3. Durasi**
- Duration in minutes
- Default: 110 minutes (1 hour 50 minutes)
- Can be adjusted: 60-180 minutes

### Step 5: Submit and Confirm
- Click "📤 Buat Jadwal Seminar" button
- System will:
  1. Fetch all matching kelompok from database
  2. Generate schedule with 4 time slots per day:
     - 08:00 - 09:50 (Slot 1)
     - 10:00 - 11:50 (Slot 2)
     - 13:00 - 14:50 (Slot 3)
     - 15:00 - 16:50 (Slot 4)
  3. Distribute kelompok round-robin across slots
  4. Save to `jadwal` table
  5. Display results with table

## 📊 Schedule Generation Rules

### Time Slots
- **Morning**: 08:00-09:50, 10:00-11:50
- **Afternoon**: 13:00-14:50, 15:00-16:50
- **Duration**: 110 minutes per slot (1 hour 50 minutes)
- **Max per day**: 4 jadwal

### Distribution Algorithm
- Round-robin across all available time slots
- If more kelompok than daily slots:
  - Day 1: Slots 1-4 filled (4 kelompok)
  - Day 2: Slots 1-4 filled (next 4 kelompok)
  - etc.
- All kelompok in same ruangan

### Database Entries
Each jadwal entry contains:
- **kelompok_id**: Which group is being tested
- **waktu_mulai**: Start time (e.g., 2026-05-15 08:00)
- **waktu_selesai**: End time (e.g., 2026-05-15 09:50)
- **user_id**: Koordinator ID (dosen yang membuat jadwal)
- **ruangan_id**: Room ID
- **KPA_id**: Category (dari dosen context)
- **prodi_id**: Program (dari dosen context)
- **TM_id**: Angkatan (dari dosen context)

## ⚙️ Technical Details

### Form HTML (Generated by Backend)
The form is generated dynamically by `JadwalSeminarTools.get_form_jadwal()`:
- Input: `id="jadwal-tanggal"` - Text input for date
- Select: `id="jadwal-ruangan"` - Dropdown populated from database
- Input: `id="jadwal-durasi"` - Number input (default 110)
- Button: `onclick="if(window.__submitJadwal){...}"` - Form submit

### Two-Stage Processing
1. **Stage 1 (input_form)**: User doesn't have jadwal_stage set
   - Backend displays form
   - Sets `jadwal_stage = "input_form"`
   
2. **Stage 2 (completion)**: User submits form
   - Backend parses inputs from message
   - Validates data
   - Generates and saves jadwal
   - Returns result table
   - Sets `jadwal_stage = "completed"`

### Message Format
User's submitted form creates message:
```
[jadwal] tanggal: 15 mei 2026 | ruangan: 1 | durasi: 110
```

Backend parses using regex:
- `tanggal[:\s]+([^|]*?)` → extracts date
- `ruangan[:\s]*(\d+)` → extracts room ID
- `durasi[:\s]*(\d+)` → extracts duration

## 🐛 Troubleshooting

| Problem | Solution |
|---------|----------|
| Form doesn't appear | Check: Did agent recognize "jadwal seminar"? Check logs for `generate_jadwal_seminar` action |
| Form appears but can't fill | Check: JavaScript loaded correctly? Check `__submitJadwal` in browser console |
| Submit button doesn't work | Hard refresh (Ctrl+Shift+R), check console for errors |
| "Tidak ada kelompok" error | No matching kelompok found. Check if:  kelompok exists for your dosen context, kelompok's prodi_id matches, kelompok's TahunMasuk_id matches |
| Date parsing fails | Check date format: "15 mei 2026" or "10 juni". Month must be Indonesian. |
| Room not found error | Check if ruangan_id is valid in database |
